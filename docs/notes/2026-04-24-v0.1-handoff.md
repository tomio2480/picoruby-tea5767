## 実機動作確認の段取り（要約）

詳細手順は [2026-04-24-pico-hw-setup.md](./2026-04-24-pico-hw-setup.md) を参照．

### Step 1: 配線

| Pico ピン | TEA5767 ピン |
|---|---|
| Pin 6（GP4 / SDA） | SDA |
| Pin 7（GP5 / SCL） | SCL |
| Pin 8（GND） | GND |
| Pin 36（3V3 OUT） | VCC |

TEA5767 の ANT 端子に 75 cm ワイヤ．プルアップはモジュール内蔵前提．

### Step 2: firmware 転送

- `firmware/app.rb` → **Pico の `/home/app.rb`**（R2P2 の自動起動位置）
- `firmware/lib/tea5767.rb` / `spectrum_scanner.rb` / `serial_emitter.rb` → **Pico の `/home/lib/`**
- R2P2 シェルの `cp` / DFU / 他のアップロード手段は R2P2 のバージョン依存

### Step 3: ブラウザで動作確認

```powershell
cd "C:\path\to\your\repo\web"
ruby -run -e httpd -- . -p 8000
```

ブラウザ（Chrome / Edge）で `http://localhost:8000/` を開き，「Pico に接続」ボタンを押す．ポート選択ダイアログで Pico の COM を選択．

### Step 4: 期待する動作

- ブラウザに `Pico 接続済み．受信中...` 表示
- Canvas に左から右へバーが立ち上がる（約 10 秒で 191 ch）
- 函館 5 局（79.4 / 80.7 / 83.6 / 87.0 / 88.8 MHz）にピーク
- 検出局テーブルに 5 行
- 500 ms 後に次スキャンが始まる（連続動作）

## 既知の不確定点

すべての主要 API は picoruby/picoruby 本体で確認済みだが，R2P2 のバージョン差で以下が問題になる可能性がある．

表 3: 想定される問題と対処

| 症状 | 原因候補 | 対処 |
|---|---|---|
| ポート選択に Pico が出ない | USB CDC が無効 / 別のドライバが占有 | R2P2 のビルドを確認．他の仮想 COM ツールを閉じる |
| `LoadError: cannot load such file` | `/home/lib/*.rb` が配置されていない / パスが違う | R2P2 シェルで `ls /home/lib/` を確認．配置先を修正 |
| I2C 通信失敗（全 RSSI = 0） | プルアップ / 配線 | R2P2 シェルで `I2C.new(unit: :RP2040_I2C0, sda_pin: 4, scl_pin: 5).scan` を実行．0x60 が返るか |
| JSON パースエラー連発 | `puts` のバッファリング | `$stdout.sync = true` を `app.rb` 先頭に追加．または `puts` 後に `$stdout.flush` |
| 局が検出されない | アンテナ / ゲイン | 75 cm ワイヤを垂直に．屋外近くで再実行 |
| 周波数が微妙にズレて局名引けない | PLL 分解能 | `web/lib/station_directory.rb` の `MATCH_TOLERANCE_KHZ` を一時的に 100 などに広げて試す |
